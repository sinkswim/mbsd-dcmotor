303/*========================================================================*



304 * Function: NIRT_ProbeSignals



305 *



306 * Abstract:



307 *	returns the latest signal values.



308 *



309 * Input Parameters:



310 *	sigindices	: list of signal indices to be probed.



311 *	numsigs		: length of sigindices array.



312 *



313 *		NOTE: (Implementation detail) the sigindices array that is passed in is delimited by a value of -1.



314 *		Thus the # of valid indices in the sigindices table is min(numsigs, index of value -1) - 1.



315 *		Reason for subtracting 1, the first index in the array is used for bookkeeping and should be copied



316 *		into the 0th index in the signal values buffer.



317 *		Also, the 1st index in the signal values buffer should contain the current timestamp. Hence valid



318 *		signal data should be filled in starting from index 2. Any non-scalar signals should be added to the



319 *		buffer in row-order.



320 *



321 * Input/Output Parameters



322 *	num			: (in) length of sigvalues buffer (out) number of values returned in the buffer



323 *



324 * Output Parameters:



325 *	value		: array of signal values



326 *



327 * Returns:



328 *	NI_OK if no error



329 *========================================================================*/



330DLL_EXPORT int32_t NIRT_ProbeSignals(int32_t *sigindices, int32_t numsigs,

480/*



481 * Functions for getting Model Information:



482 * These functions are not necessary for the model to run and used to provide a better user experience.



483 *



484 * The following functions may be called either in initialization phase,



485 * or from a background process while the simulation is running.



486 *



487 * To refer to a parameter or signal in the model, 3 unique entities are used:



488 * 1. Name (full_pathname for parameters or Path_to_SourceBlock:PortNumber for signals)



489 * 2. ID



490 * 3. A 32-bit integer index



491 *



492 * Name and ID are both character arrays and may be identical. The ID is not seen by the user and



493 * may include some coded information to make access faster.



494 *



495 * An index is an integer that is used as a key into tables for constant time access in RT.



496 * Index may have any value > 0. With -1 being the special invalid index.



497 *



498 * On RT, the parameter/signal information is extracted from the DLL using their corresponding IDs.



499 * The function should return in addition to datatype/dimension etc information, the index for the ID.



500 * This index is used subsequently when probing a signal value or setting value of a parameter.



501 *



502 * The list of IDs may be either read in from a file generated when building the DLL or another



503 * function may be exported to generate the list.



504 */



505
